# ACTIVIDAD 2.1 Desarrollo de aplicación con Intent implícito para email

**Ciclo Formativo de Grado Superior**  
**Desarrollo de Aplicaciones Multiplataforma**  
**Programación Multimedia y Dispositivos Móviles**

---

## 1. Objetivo

Demostrar conocimientos sobre desarrollo de aplicaciones para dispositivos móviles, considerando los siguientes aspectos:

- Generación de la estructura de clases necesaria para la aplicación (RA2.a).
- Establecer permisos requeridos para el funcionamiento de las aplicaciones (RA2.j).

---

## 2. Enunciado

**Individualmente**, realizad la siguiente tarea competencial evaluable para demostrar un conocimiento básico en relación con "Aplicaciones Móviles Android":

1. Cread una aplicación, tomando como referencia la de la Ilustración 1, que permita:

   - **a)** Escribir en cuadros de texto para indicar el destinatario, el asunto y el cuerpo de un correo electrónico.
   - **b)** A través de un botón enviar con un intent implícito a una aplicación de envío de correo electrónico.

   > 📱 *Ilustración 1 – Interfaz de ejemplo de App para enviar emails* *(ver imagen adjunta)*

   - **c)** La aplicación externa a la que llama el intent implícito debe mostrar la dirección, el asunto y el cuerpo del correo configurados en el apartado a), para ser enviado, como muestra el ejemplo de la Ilustración 2.

   > 📱 *Ilustración 2 – Correo listo para enviar en app externa* *(ver imagen adjunta)*

2. En un documento Word, documentad lo siguiente:

   - **a)** ¿Qué ha sido necesario hacer en el terminal emulador para poder enviar los correos a través de la aplicación de correo?
   - **b)** Documentad con capturas la interfaz de usuario y el email con los datos tomados de la interfaz y listo para ser enviado.

3. Haced un `.zip` con el proyecto de Android Studio y el documento Word y subidlo al aula virtual con el siguiente nombre:  
   `DAM2_PMM_UT2_Actividad1_Nombres-alumnos`

---

## 3. Evaluación

- La evaluación se realizará conforme a la siguiente lista de cotejo.
- La presentación de trabajos **no genuinos o plagiados** conllevará una calificación de **0 puntos** en la actividad para todos los alumnos involucrados.
- La actividad deberá entregarse antes de la fecha de cierre configurada en el aula virtual para ser evaluada. De no ser así, se puntuará con un 0. Se reabrirá el plazo de entrega para las convocatorias de Ordinaria y Extraordinaria para aquellos alumnos que deban presentarse a ellas.
- Tras la entrega, el docente podrá realizar preguntas a los alumnos para evaluar su grado de contribución a la actividad.
